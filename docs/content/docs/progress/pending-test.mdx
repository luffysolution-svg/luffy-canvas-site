# 待测试

- 生图响应待确认：模拟渠道已接收请求但浏览器未读到响应的网络错误，工作台结果卡、历史记录和画布节点都应显示「待确认」而非失败，且不提供直接重试；若图片已返回但本地保存失败，仍应显示并允许立即下载。
- 生图结果本地保存：使用返回临时图片 URL 的渠道生成图片后，刷新页面并重新打开生成记录；再删除任意画布或素材并刷新，历史图片仍应能正常显示、下载和加入参考图。
- 品牌与部署：确认网页标题、顶部品牌、GitHub/文档入口均指向 `luffy-canvas-site`，首页和顶部「致谢」入口可访问上游 `basketikun/infinite-canvas`；Netlify 生产站点首页及 `/image`、`/video`、`/canvas` 刷新后均可正常访问，公开仓库与部署产物不包含 API Key。
- AI 渠道预设与迁移：分别新增 OpenAI、New API、OpenAI 兼容、Gemini、Qwen/百炼、完全自定义渠道，确认预设协议与默认 URL 正确；旧版单能力模型和单调用脚本刷新后应自动迁移且仍可使用。删除一个仍被画布节点引用的渠道后，该节点请求应明确报“渠道已删除”，不能静默改用其他渠道。
- 渠道连接与鉴权：验证“测试连接”和“拉取模型列表”在 Bearer API Key、无需鉴权两种模式下均正确；错误 Key、错误 Base URL、限流以及浏览器 CORS 拦截应显示可诊断的中文提示。New API 和第三方中转站需分别测试 Base URL 已含 `/v1` 与未含 `/v1` 的情况。
- 原生图像与文本：验证 OpenAI 兼容生图/改图、Gemini 生图/文本、Qwen-Image 文生图/参考图编辑和 Qwen OpenAI 兼容 Responses；覆盖多张参考图、蒙版、1–15 张结果、不同尺寸与透明背景。Gemini 3.1 Flash、Flash Lite、Pro 的比例和 512/1K/2K/4K 应分别按模型收口；Qwen 编辑最多 3 张且单张 10MB，2.0 系列按总像素、Edit Max/Plus 按单边 512–2048 校验，越界时应报错而不能静默缩放。非方形结果卡片应完整显示而不被裁切。
- 原生视频：验证 OpenAI Videos、New API、Seedance、Gemini Veo 与 Wan 的任务创建、刷新后继续轮询、失败信息和结果保存；Sora 覆盖模型专属尺寸/时长和参考图居中裁剪；Veo 覆盖 16:9/9:16、4/6/8 秒、720p/1080p/4K、首帧、Veo 3.1 首尾帧插值（必须恰好 2 张图）、多参考图与近 2 天 Veo 视频续写；Wan 覆盖 T2V/I2V/R2V、北京与国际站差异、480p/720p/1080p、参考图/视频/音频、水印，以及仅 Wan2.6 Flash 可用的声音开关。
- 多能力模型与自定义脚本：同一模型同时勾选多种能力后，应出现在对应模型选择器中，且各能力脚本互不覆盖；分别验证自定义生图脚本收到 `mask`、视频脚本收到 `videos`/`audios`，无鉴权脚本不发送 Authorization，脚本语法错误不能保存，刷新后脚本视频结果仍可恢复。
- URL 快速导入：分别使用查询参数和 URL fragment 导入 `baseUrl`、`apiKey`、`provider`，确认首个渠道、协议和默认模型同步更新，导入后敏感参数立即从地址栏移除，普通 fragment 不受影响。
- 提示词来源：6 个内置来源应从 Image Prompts 统一仓库读取，更新后数量依次为 323、494、53、76、126、129；提示词仍按 6 个来源分组并可独立启用，来源内可继续按 `tags` 筛选。添加标准 JSON URL 后应能查看内容，填写非数组 JSON 或不可访问地址时应显示失败，并继续保留该来源上一次成功缓存的内容。
- 提示词来源界面：来源应以卡片列表展示，启用开关位于左侧，数量、同步状态和上次成功时间作为次级信息显示，查看、拉取及自定义来源编辑/删除操作使用带文字按钮；底部定时拉取区域应保持独立边框布局。
- 我的提示词：在提示词中心切换到「我的提示词」，验证新增、编辑、删除和搜索；从提示词库保存后应出现在个人列表、画布左侧提示词面板和提示词选择弹窗中，刷新页面后数据仍保留在本地，Agent 搜索提示词时也应能查到。
- 画布提示词库：不展开任何公共来源直接搜索其中的提示词，匹配项应自动显示；点击「插入画布」后应创建正文正确且标题保持为提示词标题的文本节点。
- 全站 Agent：新增 `generation_get_status` 工具，画布生成节点可按 `nodeIds` 查询，生图和视频工作台提交后会返回 `taskId` 并可查询排队、运行、成功或失败状态；需验证查询只由当前活动标签页返回。
- 本地 Agent 多标签页隔离：同时打开两个不同画布并连接同一个 Agent，分别聚焦标签页后通过 MCP 读取和修改画布，操作应只落在当前聚焦页面；网页面板发起的整个 Codex turn 应固定操作发起页面，即使中途聚焦另一标签页也不能切换目标；关闭当前页面后应回退到最近聚焦且仍连接的页面，其他页面回传同一请求结果应被拒绝。
- 本地 Agent 多标签页会话同步：所有标签页共享同一个站点级 Codex 活跃线程；任一页面发送消息、新建、恢复或删除会话后，其他页面应同步活跃线程和聊天记录；Agent 输出仅显示在事件所属线程，运行中不能新建、恢复、删除或再次发送任务。
- 本地 Agent 运行状态同步：在一个标签页运行较长 Codex 任务，等待某张工具卡显示「工具完成」后再打开或刷新第二个标签页；第二个标签页应立即显示 Codex 正在运行并禁用发送，整轮结束后两个标签页同时恢复；工具卡只显示「工具完成」，整轮结束由「本轮完成」表示。
- 本地 Agent 图片附件落画布：在右侧 Agent 上传参考图并要求基于商品信息创建生图流程，附件应创建为保持原比例的真实图片节点，分析提示词应创建为文本节点，二者都应连接到生成配置节点；刷新页面后参考图仍可显示并参与生成。任务中途切换到其他标签页时，附件只能写入发起任务的标签页；若发起标签页关闭，附件读取应失败且不能落入其他画布。
- Agent 对话滚动：从历史或日志切回对话、恢复其他会话时应自动定位到最新消息；手动向上浏览后应显示向下箭头且新消息不强制打断阅读，点击箭头后平滑回到底部并继续跟随新消息。
- Agent 消息区分：用户消息应在右侧以主题浅底气泡显示，AI 回复应在左侧保留 Agent 图标并使用开放式 Markdown 排版；长文本、多张附件、错误消息及浅色/深色主题下均应清晰且不溢出。
